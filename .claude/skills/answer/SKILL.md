# Answer

ユーザーの疑問に回答し、dailyノートに記録する。

## ワークフロー

1. ユーザーから疑問を受け取る（`/answer 疑問文`）
2. 疑問に対する回答を作成
3. 回答の複雑さを判断
4. dailyノートの疑問セクションに追記

## 判断基準

**dailyに直接記載する場合:**
- 説明が5行未満
- コード例が2つ以下
- 図解や表が不要

**InBoxに別ページを作成する場合:**
- 5行以上の説明が必要
- コード例が3つ以上必要
- 図解や表が必要

## 記載フォーマット

### パターンA: dailyに直接記載

```markdown
# 疑問
- [x] 疑問文？
  - **回答のキーワード**: 簡潔な説明
  - 補足ポイント
```

### パターンB: InBox別ページ作成時

dailyノートへの追記:
```markdown
# 疑問
- [x] 疑問文？
  - **概念名**: 一行の要約
  - 詳細: [[概念名]]
```

InBox/概念名.md を作成:
```markdown
# 概念名

## 概要
説明文

## コード例
コードブロック

## 関連
- 関連概念へのリンク
```

## ファイルパス

- dailyノート: `00_daily/YYYY/MM/YYYY-MM-DD.md`（今日の日付）
- InBox: `03_InBox/概念名.md`（概念名のみ、修飾語なし）

## 実行手順

1. 今日の日付からdailyノートのパスを特定
2. dailyノートを読み込み、「# 疑問」セクションを探す
3. 疑問に対する回答を作成
4. 回答の複雑さを判断
5. 複雑な場合はInBoxページを先に作成
6. dailyノートの「# 疑問」セクション末尾に追記（`- [x]` 形式）

## コード例の技術スタック

回答にコード例を含める場合:
- バックエンド: Python (FastAPI)
- フロントエンド: TypeScript

# Learn

ユーザーの学びをdailyノートに記録する。

## ワークフロー

1. ユーザーから学びの内容を受け取る
2. 内容の複雑さを判断する
3. 適切な形式でdailyノートに追記する

## 判断基準

**dailyに直接記載する場合:**
- 説明が5行未満
- コード例が2つ以下
- 図解や表が不要

**InBoxに別ページを作成する場合:**
- 5行以上の説明が必要
- コード例が3つ以上必要
- 図解や表が必要

## 記載フォーマット

### パターンA: dailyに直接記載

```markdown
# 学び
- **概念名**: 簡潔な説明
  - 補足ポイント1
  - 補足ポイント2
```

### パターンB: InBox別ページ作成時

dailyノートへの追記:
```markdown
# 学び
- **概念名**: 一行の要約
  - 詳細: [[概念名]]
```

InBox/概念名.md を作成:
```markdown
# 概念名

## 概要
説明文

## コード例
コードブロック

## 関連
- 関連概念へのリンク
```

## ファイルパス

- dailyノート: `00_daily/YYYY/MM/YYYY-MM-DD.md`（今日の日付）
- InBox: `03_InBox/概念名.md`（概念名のみ、修飾語なし）

## 実行手順

1. 今日の日付からdailyノートのパスを特定
2. dailyノートを読み込み、「# 学び」セクションを探す
3. 内容の複雑さを判断
4. 複雑な場合はInBoxページを先に作成
5. dailyノートの「# 学び」セクション末尾に追記
